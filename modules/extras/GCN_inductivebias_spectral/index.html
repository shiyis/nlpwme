Inductive bias in GCN: a spectral perspective

author: Marc Lelarge, course: dataflowr

run the code or open it in Colab

date: April 15, 2021

Here, we focus on Graph Convolution Networks (GCN) introduced by Kipf and Welling in their paper Semi-Supervised Classification with Graph Convolutional Networks. The GCN layer is one of the simplest Graph Neural Network layer defined by:

hi(ℓ+1)=1di+1hi(ℓ)W(ℓ)+∑j∼ihj(ℓ)W(ℓ)(di+1)(dj+1), h_i^{(\ell+1)} = \frac{1}{d_i+1}h_i^{(\ell)}W^{(\ell)} + \sum_{j\sim i} \frac{h_j^{(\ell)}W^{(\ell)}}{\sqrt{(d_i+1)(d_j+1)}}, hi(ℓ+1)​=di​+11​hi(ℓ)​W(ℓ)+j∼i∑​(di​+1)(dj​+1)​hj(ℓ)​W(ℓ)​,

where i∼ji\sim ji∼j means that nodes iii and jjj are neighbors in the graph GGG, did_idi​ and djd_jdj​ are the respective degrees of nodes iii and jjj (i.e. their number of neighbors in the graph) and hi(ℓ)h_i^{(\ell)}hi(ℓ)​ is the embedding representation of node iii at layer ℓ\ellℓ and W(ℓ)W^{(\ell)}W(ℓ) is a trainable weight matrix of shape [size_input_feature, size_output_feature].

The inductive bias of a learning algorithm is the set of assumptions that the learner uses to predict outputs of given inputs that it has not encountered. For GCN, we argue that the inductive bias can be formulated as a simple spectral property of the algorithm: GCN acts as low-pass filters. This arguments follows from recent works Simplifying Graph Convolutional Networks by Wu, Souza, Zhang, Fifty, Yu, Weinberger and Revisiting Graph Neural Networks: All We Have is Low-Pass Filters by NT and Maehara.

Here we will study a very simple case and relate the inductive bias of GCN to the property of the Fiedler vector of the graph. We'll consider the more general setting in a subsequent post.

Notations

We consider undirected graphs G=(V,E)G=(V,E)G=(V,E) with nnn vertices denoted by i,j∈[n]i,j \in [n]i,j∈[n]. i∼ji\sim ji∼j means that nodes iii and jjj are neighbors in GGG, i.e. {i,j}∈E\{i,j\}\in E{i,j}∈E. We denote by AAA its adjacency matrix and by DDD the diagonal matrix of degrees. The vector of degrees is denoted by ddd so that d=A1d= A1d=A1. The components of a vector x∈Rnx\in \mathbb{R}^nx∈Rn are denoted xix_ixi​ but sometimes it is convenient to see the vector xxx as a function from VVV to R\mathbb{R}R and use the notation x(i)x(i)x(i) instead of xix_ixi​.

Community detection in the Karate Club

We'll start with an unsupervised problem: given one graph, find a partition of its node in communities. In this case, we make the hypothesis that individuals tend to associate and bond with similar others, which is known as homophily.

To study this problem, we will focus on the Zachary's karate club and try to recover the split of the club from the graph of connections. The pytorch-geometric library will be very convenient.

Note that GCN are not appropriate in an unsupervised setting as no learning is possible without any label on the vertices. However, this is not a problem here as we will not train the GCN! In more practical settings, GCN are used in a semi-supervised setting where a few labels are revealed for a few nodes (more on this in the section with the Cora dataset).

The Kernighan Lin algorithm is a heuristic algorithm for finding partitions of graphs and the results below show that it captures well our homophily assumption. Indeed the algorithm tries to minimize the number of crossing edges between the 2 communities.

c1,c2 = nx.algorithms.community.kernighan_lin_bisection(G)
classes_kl = [0 if i in c1 else 1 for i in range(34)]
visualize(G, color=classes_kl, cmap="Set2")

def acc(predicitions, classes):
    n_tot = len(classes)
    acc = np.sum([int(pred)==cla for pred,cla in zip(predicitions,classes)])
    return max(acc, n_tot-acc), n_tot

n_simu = 1000
all_acc = np.zeros(n_simu)
for i in range(n_simu):
    c1,c2 = nx.algorithms.community.kernighan_lin_bisection(G)
    classes_kl = [0 if i in c1 else 1 for i in range(34)]
    all_acc[i],_ = acc(classes_kl, biclasses)

The algorithm is not deterministic but performs poorly only a small fractions of the trials as shown below in the histogram for the number of correct predictions (note there are 343434 nodes in total):

bin_list = range(17,35)

Inductive bias for GCN

To demonstrate the inductive bias for the GCN architecture, we consider a simple GCN with 3 layers and look at its performance without any training. To be more precise, the GCN takes as input the graph and outputs a vector (xi,yi)∈R2(x_i,y_i)\in \mathbb{R}^2(xi​,yi​)∈R2 for each node iii. 

import torch
from torch.nn import Linear
from torch_geometric.nn import GCNConv
import torch.nn.functional as F

class GCN(torch.nn.Module):
    def __init__(self):
        super(GCN, self).__init__()
        self.conv1 = GCNConv(data.num_nodes, 4)# no feature...
        self.conv2 = GCNConv(4, 4)
        self.conv3 = GCNConv(4, 2)

    def forward(self, x, edge_index):
        h = self.conv1(x, edge_index)
        h = h.tanh()
        h = self.conv2(h, edge_index)
        h = h.tanh()
        h = self.conv3(h, edge_index)
        return h

torch.manual_seed(12345)
model = GCN()
print(model)
GCN(
      (conv1): GCNConv(34, 4)
      (conv2): GCNConv(4, 4)
      (conv3): GCNConv(4, 2)
    )

Below, we draw all the points (xi,yi)(x_i,y_i)(xi​,yi​) for all nodes iii of the graph. The vertical and horizontal lines are the medians of the xix_ixi​'s and yiy_iyi​'s respectively. The colors are the true classes. We see that without any learning the points are almost separated in the lower-left and upper-right corners according to their community!

h = model(data.x, data.edge_index)
visualize(h, color=biclasses)

We now explain why the GCN architecture with random initialization achieves such good results.

Spectral analysis of GCN

We start by rewriting the equation (1) in matrix form:

h(ℓ+1)=Sh(ℓ)W(ℓ), h^{(\ell+1)} = S h^{(\ell)}W^{(\ell)} , h(ℓ+1)=Sh(ℓ)W(ℓ),

where the scaled adjacency matrix S∈Rn×nS\in\mathbb{R}^{n\times n}S∈Rn×n is defined by Sij=1(di+1)(dj+1)S_{ij} = \frac{1}{\sqrt{(d_i+1)(d_j+1)}}Sij​=(di​+1)(dj​+1)​1​ if i∼ji\sim ji∼j or i=ji=ji=j and Sij=0S_{ij}=0Sij​=0 otherwise and h(ℓ)∈Rn×f(ℓ)h^{(\ell)}\in \mathbb{R}^{n\times f^{(\ell)}}h(ℓ)∈Rn×f(ℓ) is the embedding representation of the nodes at layer ℓ\ellℓ and W(ℓ)W^{(\ell)}W(ℓ) is the learnable weight matrix in Rf(ℓ)×f(ℓ+1)\mathbb{R}^{f^{(\ell)}\times f^{(\ell+1)}}Rf(ℓ)×f(ℓ+1).

To simplify, we now ignore the tanhtanhtanh non-linearities in our GCN above so that we get

y=S3W(1)W(2)W(3), y = S^3 W^{(1)}W^{(2)}W^{(3)}, y=S3W(1)W(2)W(3),

where W(1)∈Rn,4W^{(1)}\in \mathbb{R}^{n,4}W(1)∈Rn,4, W(2)∈R4,4W^{(2)}\in \mathbb{R}^{4,4}W(2)∈R4,4 and W(3)∈R4,2W^{(3)}\in \mathbb{R}^{4,2}W(3)∈R4,2 and y∈Rn×2y\in \mathbb{R}^{n\times 2}y∈Rn×2 is the output of the network (note that data.x is the identity matrix here). The vector W(1)W(2)W(3)∈Rn×2W^{(1)}W^{(2)}W^{(3)}\in \mathbb{R}^{n\times 2}W(1)W(2)W(3)∈Rn×2 is a random vector with no particular structure so that to understand the inductive bias of our GCN, we need to understand the action of the matrix S3S^3S3.

The matrix SSS is symmetric with eigenvalues ν1≥ν2≥...\nu_1\geq \nu_2\geq ...ν1​≥ν2​≥... and associated eigenvectors U1,U2,...U_1,U_2,...U1​,U2​,... We can show that indeed 1=ν1>ν2≥...≥νn≥−11=\nu_1>\nu_2\geq ...\geq \nu_n\geq -11=ν1​>ν2​≥...≥νn​≥−1 by applying Perron-Frobenius theorem. This is illustrated below.

from numpy import linalg as LA

A = nx.adjacency_matrix(G).todense()
A_l = A + np.eye(A.shape[0],dtype=int)
deg_l = np.dot(A_l,np.ones(A.shape[0]))
scaling = np.dot(np.transpose(1/np.sqrt(deg_l)),(1/np.sqrt(deg_l)))
S = np.multiply(scaling,A_l)
eigen_values, eigen_vectors = LA.eigh(S)

_ = plt.hist(eigen_values, bins = 40)

But the most interesting fact for us here concerns the eigenvector U2U_2U2​ associated with the second largest eigenvalue which is also known as the Fiedler vector.

A first result due to Fiedler tells us that the subgraph induced by GGG on vertices with U2(i)≥0U_2(i)\geq 0U2​(i)≥0 is connected. This is known as Fiedler’s Nodal Domain Theorem (see Chapter 24 in Spectral and Algebraic Graph Theory by Daniel Spielman). We check this fact below both on U2U_2U2​ and −U2-U_2−U2​ so that here we get a partition of our graph in 2 connected graphs (since we do not have any node iii with U2(i)=0U_2(i)=0U2​(i)=0).

fiedler = np.array(eigen_vectors[:,-2]).squeeze()
H1 = G.subgraph([i for (i,f) in enumerate(fiedler) if f>=0])
H2 = G.subgraph([i for (i,f) in enumerate(fiedler) if -f>=0])
H = nx.union(H1,H2)
plt.figure(figsize=(7,7))
plt.xticks([])
plt.yticks([])
nx.draw_networkx(H, pos=nx.spring_layout(G, seed=42), with_labels=True)

There are many possible partitions of our graph in 2 connected graphs and we see here that the Fiedler vector actually gives a very particular partition corresponding almost exactly to the true communities!

visualize(G, color=[fiedler>=0], cmap="Set2")

To understand why the partition of Fiedler's vector is so good requires a bit of calculus. To simplify a bit, we will make a small modification about the matrix SSS and define it to be Sij=1didjS_{ij} = \frac{1}{\sqrt{d_i d_j}}Sij​=di​dj​​1​ if i∼ji\sim ji∼j or i=ji=ji=j and Sij=0S_{ij}=0Sij​=0 otherwise. We still denote by νi\nu_iνi​ and UiU_iUi​ its eigenvalues and eigenvectors.

Define the (normalized) Laplacian L=Id−SL=Id-SL=Id−S so that the eigenvalues of LLL are λi=1−νi\lambda_i=1-\nu_iλi​=1−νi​ associated with the same eigenvector UiU_iUi​ as for SSS. We also define the combinatorial Laplacian L∗=D−AL^* = D-AL∗=D−A.

We then have

xTLxxTx=xTD−1/2L∗D−1/2xxTx=yTL∗yyTDy, \frac{x^TLx}{x^Tx} = \frac{x^TD^{-1/2}L^* D^{-1/2}x}{x^Tx}\\ = \frac{y^T L^* y}{y^TDy}, xTxxTLx​=xTxxTD−1/2L∗D−1/2x​=yTDyyTL∗y​,

where y=D−1/2xy = D^{-1/2}xy=D−1/2x. In particular, we get:

λ2=1−ν2=min⁡x⊥U1xTLxxTx=min⁡y⊥dyTL∗yyTDy, \lambda_2 = 1-\nu_2 = \min_{x\perp U_1}\frac{x^TLx}{x^Tx}\\ = \min_{y\perp d} \frac{y^T L^* y}{y^TDy}, λ2​=1−ν2​=x⊥U1​min​xTxxTLx​=y⊥dmin​yTDyyTL∗y​,

where ddd is the vector of degrees.

Rewriting this last equation, we obtain

λ2=min⁡∑i∼j(y(i)−y(j))2∑idiy(i)2, \lambda_2 = \min \frac{\sum_{i\sim j}\left(y(i)-y(j)\right)^2}{\sum_i d_i y(i)^2}, λ2​=min∑i​di​y(i)2∑i∼j​(y(i)−y(j))2​,

where the minimum is taken over vector yyy such that ∑idiyi=0\sum_i d_i y_i =0∑i​di​yi​=0.

Now if y∗y^*y∗ is a vector achieving the minimum then we get the Fiedler vector (up to a sign) by U2=D1/2y∗∥D1/2y∗∥U_2 = \frac{D^{1/2}y^*}{\|D^{1/2}y^*\|}U2​=∥D1/2y∗∥D1/2y∗​. In particular, we see that the sign of the elements of U2U_2U2​ is the same as the sign of the elements of y∗y^*y∗.

To get an intuition about (6), consider the same minimization but with the constraint that y(i)∈{−1,1}y(i) \in \{-1,1\}y(i)∈{−1,1} with the meaning that if y(i)=1y(i)=1y(i)=1, then node iii is in community 000 and if y(i)=−1y(i)=-1y(i)=−1 then node iii is in community 111. In this case, we see that the numerator ∑i∼j(y(i)−y(j))2\sum_{i\sim j}\left(y(i)-y(j)\right)^2∑i∼j​(y(i)−y(j))2 is the number of edges between the two communities multiplied by 4 and the denominator ∑idiy(i)2\sum_i d_i y(i)^2∑i​di​y(i)2 is twice the total number of edges in the graph. Hence the minimization problem is now a combinatorial problem asking for a graph partition (P1,P2)(P_1,P_2)(P1​,P2​) of the graph under the constraint that ∑i∈P1di=∑j∈P2dj\sum_{i\in P_1}d_i= \sum_{j\in P_2} d_j∑i∈P1​​di​=∑j∈P2​​dj​. This last condition is simply saying that the number of edges in the graph induced by GGG on P1P_1P1​ should be the same as the number of edges in the graph induced by GGG on P2P_2P2​ (note that this condition might not have a solution). Hence the minimization problem defining y∗y^*y∗ in (6) can be seen as a relaxation of this bisection problem. We can then expect the Fiedler vector to be close to this vector of partition (P1,P2)(P_1,P_2)(P1​,P2​) at least the signs of its elements which would explain that the partition obtained thanks to the Fiedler vector is balanced and with a small cut, corresponding exactly to our goal here.

So now that we understand the Fiedler vector, we are ready to go back toi GCN. First, we check that the small simplifications made (removing non-linearities...) are really unimportant:

Recall that the signal about the communities is in the U2U_2U2​ vector so that we can rewrite it more explicitly as

Y1(i)≈a1+b1U2(i)Y2(i)≈a2+b2U2(i), Y_1(i) \approx a_1 + b_1 U_2(i)\\ Y_2(i) \approx a_2 + b_2 U_2(i), Y1​(i)≈a1​+b1​U2​(i)Y2​(i)≈a2​+b2​U2​(i),
